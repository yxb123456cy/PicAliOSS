# 轻量级图床浏览器插件（Chrome）产品需求文档（基于WXT+Vue3+TS+阿里云OSS）

# 一、产品概述

## 1.1 产品定位

一款基于WXT框架、Vue3、TypeScript开发，依托阿里云OSS实现图片存储与管理的轻量级Chrome浏览器插件。聚焦“简单、高效、轻量化”核心需求，为用户提供图片快速上传、云端存储、链接生成、历史管理、批量操作等一站式图床服务，适配个人用户（设计师、自媒体、程序员）日常图片分享、文档插图、素材管理等场景，无需切换浏览器页面，在插件内即可完成全流程操作，兼顾易用性与性能，拒绝冗余功能，降低用户图床使用门槛。

## 1.2 核心目标

1. 实现图片“一键上传+云端存储”，上传响应速度≤3秒（单张≤50MB），操作流程不超过3步；

2. 依托阿里云OSS保障图片存储安全性、稳定性，支持多种图片链接格式（直链、Markdown、HTML）快速复制；

3. 轻量化设计，插件安装包体积≤5MB，占用浏览器内存≤100MB，不影响浏览器运行速度；

4. 适配Chrome浏览器最新版本（≥110.0），支持Windows、Mac、Linux多系统，兼容WXT框架、Vue3、TS技术栈的可扩展性需求；

5. 提供简洁直观的操作界面，降低用户学习成本，无需专业技术基础即可熟练使用。

## 1.3 目标用户

- 个人设计师：需要快速上传设计稿、截图，生成可分享链接，用于作品展示、客户沟通；

- 自媒体从业者：频繁使用图片插入文章、短视频配文，需要高效管理图片素材，快速获取链接；

- 程序员：开发过程中需要上传接口截图、报错截图、演示图片，用于文档编写、团队沟通；

- 普通用户：有日常图片存储、分享需求（如朋友圈配图、社群分享），需要简单易用的图床工具，无需搭建个人服务器。

# 二、技术栈要求（强制）

1. 框架：WXT（浏览器插件开发框架），确保插件适配Chrome生态，支持热更新、模块化开发；

2. 前端：Vue3（Composition API）+ TypeScript，保证代码可维护性、类型安全，适配插件的组件化开发需求；

3. 存储：阿里云OSS，用于图片云端存储、访问控制、防盗链设置，支持图片压缩、格式转换（可选）；

4. 打包：基于WXT默认打包配置，优化包体积，确保插件安装包≤5MB；

5. 兼容性：适配Chrome浏览器≥110.0版本，兼容Windows 10+、MacOS 12+、Linux（Ubuntu 20.04+）系统；

6. 安全：通过Chrome插件安全审核，不获取用户无关隐私（仅获取上传图片、浏览器截图权限），阿里云OSS配置访问密钥加密存储，防止泄露；

7. UI组件库：指定使用PrimeVue，要求按需引入组件，优先选择轻量型组件，保证插件轻量化，同时适配插件弹窗界面，风格保持简洁、专业，与整体产品定位一致，确保Vue3+TS类型完美兼容，无样式冲突。

# 三、核心功能需求

## 3.1 基础功能：阿里云OSS配置（必做）

用户需完成OSS配置后才能使用上传功能，配置流程简洁，支持保存配置（本地加密存储，不同步云端），降低用户配置门槛。

- 配置项：AccessKey ID、AccessKey Secret、OSS Bucket名称、Bucket所在区域、自定义域名（可选，无则使用OSS默认域名）；

- 配置校验：输入后实时校验配置有效性（调用OSS接口测试连接），提示“配置成功”或具体错误信息（如密钥错误、Bucket不存在）；

- 配置管理：支持修改、重置配置，配置信息本地加密存储（使用Chrome存储API），避免明文泄露；

- 新手引导：首次打开插件时，显示简单的OSS配置步骤提示（图文结合），可跳转至阿里云OSS控制台指引页面。

## 3.2 核心功能：图片上传（必做）

支持多种上传方式，满足不同场景需求，上传过程直观，反馈及时，确保上传成功率≥99%。

- 上传方式：
        

    - 一键拖拽：从本地文件夹、桌面拖拽图片至插件弹窗，自动触发上传；

    - 点击上传：点击插件内“上传图片”按钮，打开本地文件选择器，支持多选图片（≤10张/次）；

    - 截图上传：点击插件内“截图上传”按钮，调用Chrome截图API，截取当前浏览器页面（可框选区域），截取后自动上传；

    - 剪贴板上传：复制图片（如截图后复制、本地图片复制），插件自动检测剪贴板图片，提示“是否上传剪贴板图片”，点击确认后上传。

- 上传限制：
        

    - 单张图片大小≤50MB，支持格式：JPG、PNG、GIF、WebP、SVG；

    - 批量上传≤10张/次，避免占用过多带宽，上传过程中可取消单个或全部上传任务。

- 上传反馈：
        

    - 上传中：显示实时进度条（百分比），提示“上传中...”；

    - 上传成功：提示“上传成功”，自动生成多种链接格式，高亮显示，方便复制；

    - 上传失败：提示具体失败原因（如网络异常、OSS配置错误、图片格式不支持），支持重新上传。

- 图片命名：默认使用“时间戳+原文件名”命名（避免重复），支持用户自定义命名规则（如前缀+时间戳、自定义后缀），配置后全局生效。

## 3.3 核心功能：图片管理（必做）

支持对已上传图片进行查询、预览、操作，实现图片全生命周期管理，界面简洁，操作高效。

- 图片列表：
        

    - 展示已上传图片的缩略图、文件名、上传时间、文件大小、图片链接；

    - 支持按“上传时间”（默认倒序）、“文件名”（模糊搜索）筛选、排序；

    - 分页显示：每页显示10/20/30张（可自定义），支持页码切换、快速回到顶部。

- 图片操作：

    - 预览：点击缩略图，弹出预览弹窗（支持放大、缩小、关闭）；

    - 复制链接：支持复制多种格式链接（直链、Markdown链接、HTML img标签），点击“复制”后提示“复制成功”；

    - 下载图片：点击“下载”按钮，将云端图片下载至本地默认文件夹；

    - 删除图片：点击“删除”按钮，弹出确认提示（防止误删），确认后删除OSS云端图片及本地缓存记录；

    - 批量操作：支持勾选多个图片，执行批量删除、批量复制链接操作。

- 缓存管理：本地缓存图片缩略图、链接信息，减少OSS请求次数，提升加载速度；支持手动清除缓存（解决图片更新不及时问题）。

## 3.4 辅助功能（必做）

- 链接格式设置：支持默认链接格式（如默认复制Markdown链接），可在设置中切换，无需每次上传后手动选择；

- 防盗链设置（可选）：支持配置OSS防盗链规则（如允许指定域名访问），在插件设置中直接配置，无需登录阿里云控制台；

- 快捷键设置：支持自定义常用操作快捷键（如截图上传快捷键、打开插件快捷键），提升操作效率；

- 消息通知：上传成功、失败、配置变更等操作，通过Chrome浏览器通知提示（可在设置中关闭）；

- 版本更新：插件支持自动检测更新，提示用户更新版本及更新内容，支持一键更新。

## 3.5 可选功能（根据优先级迭代）

- 图片压缩：上传前自动压缩图片（可设置压缩质量），减小图片体积，提升上传速度，节省OSS存储空间；

- 图片格式转换：上传后可将图片转换为指定格式（如PNG转JPG、WebP），生成对应格式链接；

- 文件夹分类：支持创建自定义文件夹，对图片进行分类管理（如“设计稿”“截图”“素材”），便于快速查找；

- OSS存储空间统计：显示当前Bucket已用空间、剩余空间，提示空间不足；

- 多Bucket切换：支持配置多个OSS Bucket，可快速切换，用于区分不同用途的图片（如工作、个人）。

# 四、交互设计需求

## 4.1 整体交互原则

- 轻量化：插件弹窗尺寸适中（默认宽400px、高500px），可拖拽调整大小，不遮挡浏览器主要内容；

- 简洁直观：界面分为“上传区”“配置区”“管理区”“设置区”，标签页切换，操作路径清晰，无冗余元素；

- 反馈及时：所有操作（上传、复制、删除、配置）均有明确提示，避免用户疑惑；

- 易用性：常用操作（上传、复制链接）放在显眼位置，按钮尺寸适中，支持鼠标hover提示；基于PrimeVue组件特性，统一UI样式，确保界面简洁统一，无冗余样式，适配插件小窗口展示。

## 4.2 核心交互流程

1. 首次使用流程：打开插件 → 显示OSS配置引导 → 输入配置信息 → 校验成功 → 进入上传/管理界面；

3. 图片管理流程：进入管理标签页 → 筛选/搜索图片 → 点击图片操作（预览/复制/下载/删除） → 完成操作。

## 4.3 界面布局要求

- 顶部：插件标题、关闭按钮、最小化按钮、尺寸调整按钮；

- 导航栏：上传、管理、设置、帮助4个标签页，当前页面标签高亮；

- 上传区：上传按钮、拖拽提示区域、最近上传记录（显示3条）；

- 管理区：搜索框、筛选条件、分页控件、图片列表（缩略图+操作按钮）；

- 设置区：OSS配置、链接格式、快捷键、通知、缓存管理等选项，每个选项配有简单说明；

- 帮助区：常见问题（如OSS配置失败、上传失败）、使用教程、版本信息；所有UI组件均基于PrimeVue实现，确保交互一致性。

# 五、性能需求

1. 启动速度：插件点击打开时间≤1秒，无卡顿、无白屏；

2. 上传速度：单张≤10MB图片上传时间≤2秒，单张≤50MB图片上传时间≤3秒，批量上传（10张）≤10秒；

4. 稳定性：连续上传100张图片无崩溃，长时间打开插件（≥2小时）无内存泄漏，浏览器重启后插件配置不丢失；

5. 兼容性：在Chrome最新版本及前2个版本中正常运行，无功能异常、界面错乱。

# 六、安全需求

1. 权限控制：仅申请必要权限（图片上传、截图、本地存储、浏览器通知），不获取用户浏览记录、密码等隐私信息；

2. 配置安全：阿里云OSS AccessKey ID/Secret本地加密存储（使用Chrome storage.local，加密算法AES），不明文存储、不同步至云端；

3. 上传安全：上传图片前进行简单校验（格式、大小），防止恶意文件上传；

4. 访问安全：支持OSS防盗链配置，防止图片被非法盗用；

5. 代码安全：TypeScript类型校验，避免代码漏洞，插件打包后进行安全检测，通过Chrome插件审核标准。
